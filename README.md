# Household Budget Dashboard

A Streamlit app for visualising and analysing personal bank statements, with optional [YNAB](https://www.youneedabudget.com/) integration.

The app has two modes: a **live mode** for use with your own bank data, and a **demo mode** that runs entirely on synthetic data and can be deployed publicly (e.g. on Streamlit Cloud) without exposing any personal information.

## Features

- **Overview** — balance over time and monthly spend broken down by category
- **Cashflow** — daily net and cumulative cashflow charts
- **Forecast** — linear cashflow projection with adjustable horizon (30–365 days); exclude categories or one-off transactions; save and compare scenarios
- **Categories** — spending trends by category over time
- **Merchants** — top merchants by spend; automatic subscription detection
- **Anomalies** — statistical detection of unusual transactions using IQR per merchant
- **Budgets** — budget vs. actual comparison, manually configurable or imported from YNAB
- **Merchant AI** — auto-learning categorisation using rule-based matching + a trained ML model; supports custom categories and YNAB payee suggestions
- **YNAB API Data** — raw viewer for YNAB category groups, monthly budgets, payees, and recent transactions
- **Export** — download filtered data as CSV or a one-page PDF summary report

## Live Mode

Live mode is the default. You upload your own bank statement CSV and all personalisation (merchant overrides, custom categories, budget settings, forecast scenarios) is saved to disk under `models/`.

### Prerequisites

- Python 3.10+
- Install dependencies:

```bash
pip install -r requirements.txt
```

### Running the app

```bash
streamlit run app.py
```

### Data input

Upload a bank statement CSV via the sidebar. The app expects at minimum `Date`, `Description`, `Paid out`, and `Paid in` columns — the standard export format for most UK banks (tested against Halifax FlexDirect).

### YNAB integration (optional)

Enter your YNAB Personal Access Token in the sidebar to:

- Import your budget categories and payees for smarter transaction categorisation
- Sync current-month budget targets into the Budgets tab
- Browse raw YNAB API responses in the YNAB API Data tab

You can store your token in a Streamlit secrets file to avoid entering it each time:

```toml
# .streamlit/secrets.toml
YNAB_TOKEN = "your-token-here"
```

## Demo Mode

A live demo is hosted on Streamlit Community Cloud: **[bankstatementvisualization.streamlit.app](https://bankstatementvisualization.streamlit.app/)**

Demo mode runs the full dashboard against a synthetic dataset — no real bank data required. It is designed for public deployments (e.g. Streamlit Cloud) where you want to show what the tool can do without exposing personal finances.

### What changes in demo mode

| Behaviour | Live mode | Demo mode |
|-----------|-----------|-----------|
| Data source | User-uploaded CSV | Bundled synthetic CSV |
| File uploader | Shown | Hidden |
| YNAB token input | Shown | Hidden |
| Merchant overrides | Saved to `models/` | In-memory only |
| Budget & scenario files | Saved to `models/` | Not written |
| ML model | Trained and cached | Trained in-memory only |

### Synthetic dataset

The demo data is generated by `scripts/generate_demo_data.py` and covers ~23 months (January 2024 – November 2025). It includes:

- A monthly salary payment on the 25th (±1 day jitter)
- 16 fixed direct debits (mortgage, utilities, subscriptions, etc.)
- Variable spending across 9 categories (groceries, eating out, transport, shopping, medical, giving, pets, kids, home)
- 5 deliberately large transactions to exercise the anomaly detection

The generator uses a fixed random seed (`numpy` seed 42) so the dataset is fully reproducible.

To regenerate the demo CSV:

```bash
python scripts/generate_demo_data.py
```

### Activating demo mode

Set the `DEMO_MODE` flag before running the app.

**Locally:**

```bash
DEMO_MODE=true streamlit run app.py
```

**Streamlit Cloud — add to `.streamlit/secrets.toml`:**

```toml
DEMO_MODE = "true"
```

## Project Structure

```
app.py                      # Main Streamlit application
payee_normalizer.py         # Merchant name canonicalisation (200+ rules)
utils/
  loader.py                 # CSV loading, encoding detection, data source selection
  cleaning.py               # Column normalisation, date parsing, monetary conversion
  categorization.py         # Three-stage categorisation (overrides → YNAB → keywords)
  category_mapping.py       # YNAB category → dashboard category mapping
  merchant_ai.py            # TF-IDF + Logistic Regression categoriser and override learning
  subscriptions.py          # Recurring/subscription detection (IQR heuristic)
  anomalies.py              # Per-merchant anomaly detection (IQR)
  budgets.py                # Budget history load/save and YNAB budget import
  budget_settings.py        # Budget settings editor UI component
  forecasting.py            # Cashflow time series and mean-net forward projection
  scenario_storage.py       # Forecast scenario persistence
  charts.py                 # Plotly chart builders (balance, cashflow, categories, merchants)
  charts_forecast.py        # Forecast chart with scenario overlay
  pdf_report.py             # One-page PDF summary via ReportLab
  ynab_api.py               # YNAB REST API client
  demo_mode.py              # Demo mode detection and behaviour switches
scripts/
  generate_demo_data.py     # Synthetic bank statement generator
data/
  demo_transactions.csv     # Pre-generated synthetic dataset
models/                     # Persisted overrides, budgets, and forecast scenarios (live mode)
```
